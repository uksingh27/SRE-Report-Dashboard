SRE Report Analyzer - Process Description

This tool analyses excel files sent by SRE Team to CSOC for faster detection of malicious activities within Business Units (BUs).

CSOC mailbox receives various reports in Excel file via email every day from SRE Platform team. CSOC engineer analyses all the reports as per the process. It has been observed that the manual processing takes several hours if not days.

To minimize the processing time on analysing the SRE reports, this tool has been developed to enable faster detection of malicious activities within Business Units (BUs).

Process Summary

Based on the information provided in Guide to performing checks on reports sent by SRE team, this tool performs following three tasks:

This task breaks down the actor's email address found in the UserManager tab of the excel file into two components. The domain component gets compared with the suspicious domains listed in the static excel file, "SuspiciousDomains.xlsx" embedded in the program. After upload of the original report, the file gets analysed after clicking on "Analyze All" button. Post analysis, the results can be seen by clicking the respective navigation menus. For example, by clicking on "Suspicious Domain Analysis", it will display two tables: (1) Suspicious Activities and (2) Safe Activities tables.

This task analyses information provided in the AccessKeyManagement tab of the uploaded excel file. If it finds more than 10 access keys created in a Business Unit, it reports them in the result table. However, there are some business units given an exception, i.e. it does not report if the tenant name is matched to the tenant name provided in the "TenantExceptionList.xlsx" static file embedded in the program

This task analysis information provided in the UserEmailDomainStats tab of the uploaded excel file. If the "CHANGED_TO_VALUE" contains domains that match the suspicious domains listed in the in the embedded excel file, "SuspiciousDomains.xlsx", it reports the results in a table named "Suspicious Domains Activities" followed by another table named "Safe Email Domain Activities"
